# 📘 Software Engineering - Verification, Validation, Quality & Design Patterns

---

# 🔹 Verification & Validation

## ✅ Verification
👉 **"Are we building the product right?"**

- Ensures the software follows the specifications  
- Checks if code and design match requirements  

📌 Example:  
If the requirement is to build a program that adds two numbers  
→ Verification ensures correct implementation  

---

## ✅ Validation
👉 **"Are we building the right product?"**

- Ensures the software meets user needs  

📌 Example:  
User needs a full calculator  
But only addition is implemented  
→ Validation fails ❌  

---

## 🔹 Key Differences

| Verification | Validation |
|-------------|-----------|
| Building the product right | Building the right product |
| Focus on design & code | Focus on user needs |
| Before execution | After execution |

---

## 🔹 V&V Process

- Continuous across all development stages  

### 🎯 Objectives:
- Detect defects  
- Ensure usability  

⚠️ Software must be **good enough**, not perfect  

---

# 🔹 Software Quality

## ✅ What is Quality?

Quality = characteristics or attributes of a product  

### Types:
- **Quality of Design** → requirements & design  
- **Quality of Conformance** → implementation  
- **User Satisfaction** → quality + delivery + cost  

---

## 🔹 Views of Quality

- User view → meets user goals  
- Product view → features & functions  
- Value-based view → worth the cost  

---

## 🔹 Software Quality Factors

### McCall’s Model

#### Product Operation:
- Correctness  
- Reliability  
- Usability  
- Efficiency  
- Integrity  

#### Product Transition:
- Portability  
- Reusability  
- Interoperability  

#### Product Revision:
- Maintainability  
- Flexibility  
- Testability  

---

## 🔹 ISO 25010 Model

### Quality in Use:
- Effectiveness  
- Efficiency  
- Satisfaction  
- Freedom from risk  

### Product Quality:
- Functionality  
- Performance  
- Security  
- Reliability  
- Maintainability  

---

## 🔹 Cost of Quality

- Prevention → planning & training  
- Appraisal → testing & reviews  
- Internal failure → fixing bugs  
- External failure → customer issues

# 🔹 Pattern-Based Design

## ✅ What is a Design Pattern?

A **design pattern** is a reusable solution that describes:
- **Context** → where the problem exists  
- **Problem** → what needs to be solved  
- **Solution** → how to solve it  

---

## 🔹 Characteristics of Good Patterns

- Solve real problems  
- Proven solutions  
- Not obvious  
- Show relationships between components  
- Elegant and reusable  

---

## 🔹 Types of Patterns

### 1. Architectural Patterns
- High-level system structure  

### 2. Data Patterns
- Data-related problems  

### 3. Component (Design) Patterns
- Subsystems and components  

### 4. Interface Patterns
- UI/UX problems  

### 5. Web & Mobile Patterns
- Web applications  
- Mobile apps  

---

## 🔹 Pattern Categories (OOP)

### Creational Patterns
- Object creation  
- Example: Factory, Builder  

### Structural Patterns
- Organizing classes  
- Example: Adapter  

### Behavioral Patterns
- Object interaction  
- Example: Command, Chain of Responsibility  

---

## 🔹 Frameworks

- A **framework** is a reusable structure (skeleton)  
- Contains plug points (hooks) to customize behavior  

---

## 🔹 Pattern Description

Each pattern includes:
- Name  
- Problem  
- Context  
- Solution  
- Consequences  
- Implementation  

---

## 🔹 Pattern-Based Design Process

1. Understand requirements (big picture)  
2. Identify patterns  
3. Start with high-level patterns  
4. Refine with lower-level patterns  
5. Adapt patterns to your system  

---

## 🔹 Design Tasks

- Analyze requirements  
- Identify problem hierarchy  
- Select appropriate patterns  
- Refine design iteratively  

---

## 🔹 Common Mistakes

- Choosing wrong pattern  
- Forcing pattern into problem  
- Ignoring constraints  
- Not adapting pattern  

---

## 🔹 Anti-Patterns (Bad Practices)

- Blob → one huge class  
- Spaghetti Code → messy structure  
- Copy-paste programming  
- Silver bullet assumption  

---

## 🔹 UI Patterns Examples

- Navigation menus  
- Page layout  
- Shopping cart  
- Search systems  

---

## 🔹 Mobile Patterns Examples

- Sign-up flows  
- Maps integration  
- Notifications (popovers)  

---

# 🔥 Final Summary

- Verification → build the product right  
- Validation → build the right product  
- Quality → deliver value + reliability  
- Patterns → reusable design solutions  

✔️ To build great software:
- Use testing & inspections  
- Apply quality standards  
- Use design patterns effectively
