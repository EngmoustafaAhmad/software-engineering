# Functional vs Non-Functional Requirements

## ✅ Functional Requirements

Functional requirements describe **what the system should do** — its behavior and features.

They define the **functions, inputs, outputs, and interactions**.

### Examples:

* User can log in with username and password
* System sends email confirmation after registration
* App allows creating and editing posts
* Search returns relevant results

👉 Think: **“features of the system”**

---

## ⚙️ Non-Functional Requirements (NFRs)

Non-functional requirements describe **how the system behaves**, not what it does.

They define **quality attributes and constraints**.

### Examples:

* Performance: system responds within 2 seconds
* Security: data must be encrypted
* Scalability: supports 10,000 users simultaneously
* Usability: interface is easy to use
* Reliability: 99.9% uptime

👉 Think: **“quality of the system”**
